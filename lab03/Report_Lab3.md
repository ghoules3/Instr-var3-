## Введение  
Целью работы является построение сквозного ML-пайплайна в Apache Spark MLlib для задачи классификации пола пользователя по данным тренировок.  
В рамках варианта 3 необходимо:  
1. Выполнить подготовку и очистку данных  
2. Создать признаки:  
   -`hr_length` — длина массива пульса как прокси длительности/выносливости тренировки  
   -`avg_speed` — средняя скорость  
3. Обучить модель классификации `DecisionTreeClassifier`  
4. Оценить качество модели по метрике ROC-AUC  
5. Определить, какой признак важнее для определения пола пользователя: скорость или выносливость  
Бизнес-кейс работы связан с использованием тренировочных данных в Fitness-продуктах для сегментации пользователей, персонализации тренировочных рекомендаций и более точного таргетинга маркетинговых предложений.  
  
## Исходные данные  
В работе использовался датасет `endomondoHR.json`, содержащий данные о тренировках пользователей.  
В датасете присутствуют следующие поля:  
-`gender` пол пользователя  
-`sport` вид активности  
-`heart_rate` массив значений пульса  
-`speed` массив значений скорости  
-`altitude`, `latitude`, `longitude`, `timestamp`, `userId` и другие поля  
Файл датасета не включён в репозиторий из-за большого объёма. В работе использовалась локальная копия файла в виртуальной машине.

## Построение ML Pipeline  
Для решения задачи классификации был использован `DecisionTreeClassifier`.  
ML Pipeline включал следующие этапы:  
1. `StringIndexer` преобразование `gender` в числовую целевую переменную `label`  
2. `VectorAssembler` объединение признаков `hr_length` и `avg_speed` в единый вектор `features`  
3. `DecisionTreeClassifier` обучение дерева решений  
После этого данные были разделены на обучающую и тестовую выборки:  
-Train: **39593**  
-Test: **9862**

## Оценка базовой модели  
Для базовой модели были рассчитаны следующие метрики:  
-**ROC-AUC = 0.5000**  
-**Accuracy = 0.9269**  
-**F1-score = 0.8917**  
Несмотря на высокую Accuracy, дальнейший анализ показал, что базовая модель выродилась в предсказание только одного класса - доминирующего класса `male`.  
Это подтверждается:  
-ROC-AUC = 0.50, что соответствует случайному классификатору  
-матрицей ошибок / распределением предсказаний, где модель фактически не различает второй класс  
Таким образом, высокая Accuracy в данном случае является следствием сильного дисбаланса классов, а не реального качества модели.

Полученные результаты показывают:

-длина массива пульса, используемая как прокси выносливости, играет значительно меньшую роль  
-различия по полу в данном наборе данных проявляются прежде всего в интенсивности движения, а не в длительности записи пульса
