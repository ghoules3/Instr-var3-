## Описание задания  
В рамках практической работы необходимо построить ML-пайплайн в Apache Spark MLlib для задачи классификации пола пользователя по данным тренировок.  
Для **варианта 3** требуется:  
1. Выполнить подготовку данных  
2. Создать признаки:  
   -`hr_length` длина массива пульса (прокси длительности/выносливости тренировки)  
   -`avg_speed` средняя скорость  
3. Обучить модель классификации **DecisionTreeClassifier**  
4. Оценить качество модели по метрике **ROC-AUC**  
5. Определить, какой признак важнее для определения пола: **скорость** или **выносливость**  
  
## Структура папки  
-`lab_03_var3.ipynb` основной ноутбук с кодом  
-`Report_Lab3.md` отчёт по выполнению работы  
-`screenshots/` скриншоты результатов  
-`Readme.md` описание работы  
  
## Данные  
Используется датасет **`endomondoHR.json`**.  
Файл датасета не включён в репозиторий из-за большого размера.  
В работе использовалась локальная копия файла в виртуальной машине.  
  
## Основные этапы работы  
1. Загрузка JSON-датасета в Spark  
2. Анализ структуры данных и пропусков  
3. Очистка данных:  
   -исключение записей с `gender = unknown`  
   -удаление строк без массива скорости  
4. Формирование признаков:  
   -`hr_length`  
   -`avg_speed`  
5. Построение ML Pipeline:  
   -`StringIndexer`  
   -`VectorAssembler`  
   -`DecisionTreeClassifier`  
6. Разделение данных на обучающую и тестовую выборки  
7. Оценка модели по метрикам ROC-AUC, Accuracy и F1-score  
8. Балансировка обучающей выборки  
9. Анализ важности признаков и бизнес-интерпретация результатов  
  
## Основные результаты  
Базовая модель на несбалансированной выборке вырождалась в предсказание доминирующего класса, поэтому при высокой Accuracy имела низкую диагностическую ценность.  
После балансировки обучающей выборки качество модели улучшилось:  
-**ROC-AUC = 0.6409**  
-**Accuracy = 0.7907**  
-**F1-score = 0.8332**  
Анализ важности признаков показал:  
-`avg_speed` **0.944598**  
  -`hr_length` **0.055402**  
  
## Вывод по варианту  
Для определения пола пользователя в данной модели более важным признаком является **скорость**, а не **выносливость**.
